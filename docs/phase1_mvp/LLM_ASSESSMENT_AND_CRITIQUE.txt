## Project Overview

**sucurindex** is a CLI-based file indexing and organization system. It provides a virtual file index with metadata storage, tagging, linking, and synchronization capabilities.

## Project Structure

```
sucurindex/

├── tests/

└── metadata/               # Storage directory for TOML entries
```

### 3. `src/file_system.py`

File system operations - metadata reading, listing, sync checking, directory traversal.

**Functions:**

- `read_file_metadata(path: str) -> dict` - Reads file stats. Returns `{"mtime": float, "size": int}`. Raises `FileNotFoundError` if file doesn't exist.

- `list_file_entries() -> list[FileEntry]` - Returns all indexed FileEntry objects from the metadata folder. Reads all `.toml` files.

- `check_entries() -> dict[str, list[FileEntry]]` - Sync check. Returns `{"changed": [], "missing": [], "unaltered": []}` by comparing filesystem state to indexed entries.

- `slither(rootpath: str) -> list[str]` - Recursively walks directory tree, returns list of all file paths (absolute).

### 4. `src/parsing.py`

TOML serialization and filtering operations.

**Functions:**

- `read_toml_file_entry(id: str, skip_path_validation: bool=False) -> FileEntry` - Loads a FileEntry from its TOML file. Raises `FileNotFoundError` if not found.

- `write_toml_file_entry(file_entry: FileEntry) -> str` - Serializes FileEntry to TOML string format.

- `filter_single_attribute(universe: list[FileEntry], attr: str, term, debug: bool=False) -> list[FileEntry]` - Filters entries where attribute equals term (exact match).

- `filter_list_attribute(universe: list[FileEntry], attr: str, term, debug: bool=False) -> list[FileEntry]` - Filters entries where term is in the list attribute (e.g., tags, links).

- `update_file_entry(file_entry: FileEntry, attributes: dict) -> FileEntry` - Creates new FileEntry with updated attributes, preserving unmodified fields.

- `update_list_attribute(attr: list, item, mode) -> list` - Adds (`mode="a"`) or removes (`mode="r"`) item from list attribute. Raises Exception for invalid mode.

### 7. `src/cli.py`

CLI command-line interface - argument parsing and command dispatch.

**Functions:**

- `main()` - Entry point. Sets up argparse with subcommands: add, remove, list, read, sync, edit, slither. Parses args and dispatches to command handler.

- `cmd_add(path: str, name: str, description: str, tags: list[str], links: list[str])` - Adds file to index. Checks for duplicates, creates FileEntry, writes TOML.

- `cmd_remove(id: str)` - Removes index entry. Prompts for confirmation before deletion.

- `cmd_list(id, name, description, tags, lists)` - Lists indexed files. Supports filtering by any attribute (single or list).

- `cmd_read(id: str)` - Displays full details of a single indexed file.

- `cmd_sync(hard: bool=False)` - Checks indexed files against filesystem. Reports missing, changed, and OK files. If `--hard`, updates metadata for missing/changed entries.

- `cmd_edit(id: str, name: str=None, description: str=None, tags: list[str]=None, links: list[str]=None, atags: list[str]=None, rtags: list[str]=None, alinks: list[str]=None, rlinks: list[str]=None)` - Edits file metadata. Supports full replacement or incremental add/remove for tags and links.

- `cmd_slither(path: str)` - Recursively indexes all files in a directory tree. Calls `cmd_add` for each file found.

**CLI Commands Summary:**

| Command | Description |
|---------|-------------|
| `add` | Add file to index |

| `list` | List/search indexed files |
| `read` | Show file details |

## Features Implemented (MVP)

1. **Add files to index** - File path validation, UUID generation, metadata capture
2. **TOML storage** - One `.toml` file per indexed entry
3. **List/Search** - Filter by name, description, tags, links
4. **File details** - Full metadata display
5. **Sync** - Detect missing files, detect size/mtime changes
6. **Link management** - Add/remove links between indexed files
7. **Tag/description editing** - Full and incremental updates
8. **Delete metadata** - Remove index entries
9. **Config file** - Configurable storage directory
10. **CLI** - Complete command-line interface
11. **Logging** - Function call tracing with timestamps

## Data Model

```
FileEntry:
  - id: UUID
  - path: str (absolute)
  - name: str
  - description: str
  - tags: list[str]
  - links: list[UUID]
  - mtime: float (Unix timestamp)
  - size: int (bytes)
  - missing: bool
```

## Storage

- TOML files in configurable directory (default: `./metadata/`)
- Config in `config.toml` with `Persistent Storage.Primary Storage Directory` key
- Logs in `./logs/suci_TIMESTAMP.log`
