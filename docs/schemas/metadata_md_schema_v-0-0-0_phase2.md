# MbdoiDown MarkDown Dialect and metadata encoding Schema v0.0.0

## Description

- MboiDown stands for: MboiDown is a Brazilian Object Indexing Directional
Ontology using Wikilink Notation
- Mboi (or, more often, mbói and mbóia) means snake in native brazilian
languages belonging to the Tupi-Guarani grouping.
- MboiDown is abreviated as MD and files writen in MboiDown are marked with the
".md" filename extension
- It defines a Markdown-compatible file structure and syntax used for
representing, metadata and directional, semantic relationships indexed into
a knowledge graph by SucurIndex
- It is simple to parse, human-friendly and offers great integration with the
native styles of wiki- and knowledge-organization-software
- It is writen in based in ComonMark style, with additional support for
wikilinks and adherence to some specific rules, described bellow

## Extra rules

### Rule 1
A MboiDown v0.0.0 file stores the metadata of one, and only one, taxon. A taxon
is the indexed representation of an entity/object (denominated Artifact) which
is understood by the SucurIndex software

### Rule 2
A MboiDown v0.0.0 file has its filename in the format
"hhhhhhhh-hhhh-hhhh-hhhh-hhhhhhhhhhhh.md", where h represents a hexadecimal
algorism and the string preceding ".md" represents a 32-byte uuid, which is the
id (a metadata class assigned by SucurIndex) of the taxon stored in that file.

### Rule 3
A MboiDown v0.0.0 file has exactly line which is a level one markdown header.
This line is the first non-blanck line of the document and is written strictly
in the format "# NAME\n", where "\n" is a newline charcter, NAME represents a
string containing only the characters in at least one of the sets
{a-z,A-Z,0-9,-,_, ,(,),.,;,:,&,%} or {,}, whose last character is not a
whitespace and matches exactly the name (a metadata class assigned by
SucurIndex) of the taxon stored in that file

### Rule 4
A MboiDown v0.0.0 has exactly one occurence of the string
"\n---\n## --- SucurIndex Metadata ---\n\nMboiDown v0.0.0\n\n---\n", which is
denominated "SucurIndex metadata block delimiter"

### Rule 5
The content in between the strings described by Rules 3 and 4 is a string
written in strict ComonMark MarkDown style with the added restriction
(pleonasticaly here stated) that it must not violate rules 2, and 3, and, thus,
must contain no level 1 header and no occurences of the SucurIndex metadata
block delimiter. This content exactly matches the description (a metadata class
assinged by SucuIndex) of the taxon stored in that file, should that taxon have
a description, and is the "" string, should said taxon not have a description.

### Rule 6
The SucurIndex metadata block delimiter is followed by a stirng formated
exactly as "SUCURINDEXMETADATABLOCK\n---", which is followed by no non-blanck
line and where SUCURINDEXMETADATABLOCK is a string denomidated "SucurIndex
metadata block", whose characteristics and purpose are ostensibly described and
determined by the following rules

### Rule 7
The SucurIndex metadata block contains at least one line that is a level 1
MarkDown header and at least one line that is not so.

### Rule 8
All lines belonging to the SucurIndex metadata block which are level 1 MarkDown
headers are written sctritly following the format "# CLASSNAME", being
CLASSNAME a string which exactly matches the canonical name of a SucurIndex
metadata class. All SucurIndex metadata classes of the taxon represented by the
file are present in the SucurIndex metadata block, excluding its name and
description (and, thusly, including its id)

### Rule 9
All lines belonging to the SucurIndex metadata block which are not level 1
MarkDown headers are generated by the concatenation of the string correspondent
to the one generated by the appending of a newline character to the the
canonical string representation of each metadata instance belonging to the
SucurIndex metadata class whose name exactly matches the string produced
by the exclusion of the first three characters from the immediatly
preceding line which is a level 1 MarkDown header.

### Rule 10
These rules are denominated "The ten commandments of MboiDown version 0.0.0"
